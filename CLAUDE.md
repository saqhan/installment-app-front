# Installment App — Frontend (Angular)

## Контекст проекта
SPA для учёта продаж в рассрочку для инвесторов.

## Роли
- **Владелец** — полный доступ ко всему
- **Инвестор** — только свои сделки, своя доля, свои клиенты
- **Сотрудник** — ввод продаж, мониторинг, без финансовой разбивки

## Стек
- Frontend: Angular (этот проект)
- Backend: Go REST API (отдельный репозиторий, ещё не готов)
- Деплой: Vercel (фронт), Railway (бэк + PostgreSQL)
- Auth: JWT

## MVP — что делаем сейчас (Фаза 1)
1. Структура проекта: модули, роутинг, окружения dev/prod
2. Auth модуль: страница логина, регистрация по invite-токену
3. AuthGuard: защита роутов, редирект на /login
4. HTTP интерцептор: автоматически добавляет Authorization header
5. Заглушки для основных страниц (клиенты, сделки, дашборд)

## Следующие фазы
- Фаза 3: клиенты, сделки, калькулятор рассрочки, график платежей
- Фаза 4: просрочки, кабинет инвестора

## Договорённости
- API пока недоступен — использовать моки/заглушки через Angular сервисы
- Все защищённые роуты проверяют JWT из localStorage
- RBAC на уровне роутов (CanActivate) и компонентов (*ngIf по роли)
- CI/CD: GitHub Actions → Vercel при push в main
